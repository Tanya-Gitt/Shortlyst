# Contributing to Shortlyst

Thank you for your interest in contributing! Here's how to get started.

## Development Setup

1. Fork and clone the repo
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Install dependencies (see README)
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Code Style

- **Python:** Follow PEP 8 (use `black` for formatting)
- **JavaScript/React:** Use ESLint and Prettier
- **Commits:** Use clear, descriptive messages

## Reporting Bugs

Include:
- Exact error message
- Resume file format (PDF/DOCX/TXT)
- Steps to reproduce
- Expected vs actual behavior

## Feature Requests

Describe the use case and why it's valuable. Check existing issues first to avoid duplicates.

## Pull Request Process

1. Update README if needed
2. Add tests for new features
3. Ensure all tests pass
4. Write a clear PR description

## Questions?

Open an issue or reach out to the maintainer.
